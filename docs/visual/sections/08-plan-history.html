8. Plan History vs Current Plan Logic

📋 Šta je ovo?

Plan History vs Current Plan Logic pokazuje kako sistem određuje koji plan je aktivan na osnovu datuma. Ovo je decision tree koji određuje šta klijent vidi u aplikaciji.

Ključni pojmovi:

  • planHistory[] - Niz svih dodeljenih planova (prošlih i budućih)
  • currentPlanId - ID plana koji je klijent otključao i platio
  • Active plan - Plan čiji je planStartDate <= today <= planEndDate
  • Backward compatibility - Ako klijent ima samo currentPlanId (stari sistem), migrira se u planHistory

💡 Primer iz prakse

Scenario: Klijent Petar ima 3 plana u planHistory: jedan prošli (završen), jedan aktivan (trenutni), i jedan budući (zaključan).

Šta se dešava:

  1. Sistem proverava planHistory i traži plan gde je planStartDate <= today <= planEndDate
  2. Pronalazi aktivan plan i proverava da li je to plan koji je Petar otključao (currentPlanId)
  3. Ako jeste, vraća plan sa planStatus: 'current' (Petar može da trenira)
  4. Ako nije, vraća plan sa planStatus: 'future' (Petar mora da otključi)
  5. Ako nema aktivnog plana, vraća null ili najnoviji prošli plan

Rezultat: Petar vidi jasno koji plan je aktivan, koji je zaključan, i koji je završen.

🔍 Detaljno objašnjenje

Zašto je ovo važno? Plan History vs Current Plan logika određuje šta klijent vidi u aplikaciji i da li može da trenira.

Kako radi logika:

  • Plan History provera: Sistem prvo proverava da li klijent ima planHistory
  • Active plan lookup: Traži plan gde je planStartDate <= today <= planEndDate
  • Backward compatibility: Ako klijent nema planHistory ali ima currentPlanId (stari sistem), migrira se u planHistory
  • Null handling: Ako nema aktivnog plana, vraća null ili najnoviji prošli plan za pregled

Active plan kriterijum: Plan je aktivan samo ako je planStartDate <= today <= planEndDate. Ako je planEndDate prošao, plan je završen (previous). Ako je planStartDate u budućnosti, plan je zaključan (future).
